# Anchors Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to
> implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Replace kern's single root "purpose" with multiple named **anchors**
(top-level children of the root) plus a permanent **generic** catch-all, and
auto-promote dense generic clusters to anchors.

**Architecture:** The root becomes a pure dispatcher. Each anchor *is* a named
child kern carrying the per-node routing vector (`anchor_vec`). An incoming
entity matches the best anchor whose acceptance probability clears a floor; non
-matchers fall to the `generic` child. `purpose_*` is renamed to `anchor_*` in
place — bincode is positional, so the live graph is unaffected.

**Tech Stack:** Rust, bincode (positional persistence), rayon, clap, custom
kern_rpc transport.

**Data-safety invariant (do not violate):** Keep `anchor_text`/`anchor_vec` at
the SAME struct field positions `purpose_text`/`purpose_vec` occupied. Bincode
stores no field names; reordering or changing the field SET corrupts every
existing `.kern` shard.
